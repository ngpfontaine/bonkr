Sections #

Structural basics in BONKR are focused on <main> content.

<header>, <nav>, <footer>, & <aside>are left alone to avoid "Bootstrapping" your styles. When adding these, you can use the flex helper classes below; or just write custom styling.

Each section on this page lies within a (you guessed it) <section>, which are centered, with padding, & max-width. 

section {
  margin: 0 auto;
  max-width: 1200px;
  padding: 1.76rem 2.35rem; }

You can overwrite with your preferred <max-width>, or use the max-width helper classes below.

Articles #

An <article> is used to make a bar like this. You can nest a <section> inside to — center, add padding, & set max-width (per above). This is helpful when you want a background-colour, background-image, or width: 100% content. 

<article>
  <section>
    <p>
      I am an article
    </p>
  </section>
</article>
Why?

This is done to avoid unecessary classes on divs like .bar & .inner and such. Yes it's opinionated, and yes it's 3 lines of code you can remove if you so choose. 

article {
  margin: 3.52rem 0;
  background: #f2f2f2; }

Dialog Modal #

Each button below has dialog-btn-open class and a data-dialog-id attribute that corresponds to a dialog modal's #dialog-<id>.

Basic setup would be: 

<button class='dialog-btn-open' data-dialog-id='0'>Button Will Open Dialog<button>

<div class='dialog' id='dialog-0>
  <div class='dialog-outer'> // Overlay to cover content & close via click
    <div class='dialog-inner'> // Display dialog content
      <button class='dialog-btn-close'>X to Close Dialog</button>
      // Content..
    </div>
  </div>
</div>

Padding

Most semantic elements already have default margins & padding (in rem) for base styling, but anything can be customized further.

  • Basic setup is .pd- + direction + number
  • Directions: a - all, t - top, r - right, b - bottom, l - left
  • Numbers: 0-8
    • Which equals 0.3rem - 2.35rem
    • Or about 5-40px at body: 17px

.pd-a-3 .pd-t-3 .pd-r-5 .pd-b-2 .pd-l-7

Additionally, you can use .pd-ch-# to give direct children the same padding.

Margin

  • Same as Padding, just using .mg-
  • Can also use .mg-0-auto for centering with margin: 0 auto;
.mg-a-2
.mg-t-3

Parent Container Classes

Define parent container as display flex, then add classes to customize or define children 


  .d-flx { display: flex; }
  .flx-row { flex-direction: row; }
  .flx-col { flex-direction: column; }

  .flx-wr-n { flex-wrap: nowrap; }
  .flx-wr-w { flex-wrap: wrap; }
  .flx-wr-r > * { flex-wrap: wrap-reverse; }

  .flx-spc-1 { margin-left: -0.3rem; margin-right: -0.3rem; }
  .flx-spc-2 { margin-left: -0.59rem; margin-right: -0.59rem; }
  .flx-spc-3 { margin-left: -0.88rem; margin-right: -0.88rem; }
  .flx-spc-4 { margin-left: -1.18rem; margin-right: -1.18rem; }
  
  .flx-children-0  > * { flex: 0; }
  .flx-children-1 > * { flex: 1; }
  
  .flx-children-g-0 > * { flex-grow: 0; }
  .flx-children-g-1 > * { flex-grow: 1; }
  .flx-children-s-0 > * { flex-shrink: 0; }
  .flx-children-s-1 > * { flex-shrink: 1; }
  
  .flx-children-spc-1 > * { margin: 0 0.3rem 0.6rem; }
  .flx-children-spc-2 > * { margin: 0 0.59rem 1.18rem; }
  .flx-children-spc-3 > * { margin: 0 0.88rem 1.76rem; }
  .flx-children-spc-4 > * { margin: 0 1.18rem 2.36rem; }
